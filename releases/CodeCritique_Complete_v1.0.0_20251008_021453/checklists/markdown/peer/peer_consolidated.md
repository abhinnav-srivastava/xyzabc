# Self Review

## Code Review

- **MUST:** Logic Review
  - **How to Measure:** Review the business logic for correctness
  - **How to Measure:** Trace through the logic flow
  - **Rule Reference:** REV-001
  - **Additional Info:** Check edge cases

- **GOOD:** Performance Review
  - **How to Measure:** Check for performance issues and optimizations
  - **How to Measure:** Analyze time complexity
  - **Rule Reference:** REV-002
  - **Additional Info:** Consider Big O notation

## Functionality Review

- **GOOD:** Verify requirements implementation
  - **How to Measure:** Ensure all user stories and acceptance criteria have been properly implemented.
  - **How to Measure:** Review against user story acceptance criteria

- **GOOD:** Check business logic correctness
  - **How to Measure:** Verify that business logic is implemented correctly and handles all business rules.
  - **How to Measure:** Unit tests and integration tests

## Style & Naming Review

- **GOOD:** Verify naming conventions
  - **How to Measure:** Ensure all variables, functions, and classes follow consistent naming conventions.
  - **How to Measure:** Code review and linting tools

- **GOOD:** Check code formatting
  - **How to Measure:** Verify that code formatting follows project standards and is consistent throughout.
  - **How to Measure:** Run formatter and check for violations
