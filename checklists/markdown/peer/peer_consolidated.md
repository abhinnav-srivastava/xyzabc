# Self Review

## Code Review

- MUST: Logic Review - Review the business logic for correctness
  - How to Measure: Trace through the logic flow
  - Rule Reference: REV-001
  - Additional Info: Check edge cases

- RECOMMENDED: Performance Review - Check for performance issues and optimizations
  - How to Measure: Analyze time complexity
  - Rule Reference: REV-002
  - Additional Info: Consider Big O notation

## Functionality Review

- GOOD: Verify requirements implementation - Ensure all user stories and acceptance criteria have been properly implemented.
  - How to Measure: Review against user story acceptance criteria

- GOOD: Check business logic correctness - Verify that business logic is implemented correctly and handles all business rules.
  - How to Measure: Unit tests and integration tests

## Style & Naming Review

- GOOD: Verify naming conventions - Ensure all variables, functions, and classes follow consistent naming conventions.
  - How to Measure: Code review and linting tools

- GOOD: Check code formatting - Verify that code formatting follows project standards and is consistent throughout.
  - How to Measure: Run formatter and check for violations
