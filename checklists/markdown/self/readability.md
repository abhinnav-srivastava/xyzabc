# Code Review Checklist

- Code is well-commented and self-documenting (GOOD)
  - Complex logic explained
  - Function purposes documented
  - Business rules clarified
  <!-- Code should be easy to understand for future maintainers -->

- Variable and function names are descriptive (MUST)
  - Names reflect purpose
  - No abbreviations
  - Consistent naming conventions
  <!-- All identifiers should clearly indicate their purpose -->

- Functions are focused and not too long (GOOD)
  - Single responsibility principle
  - Reasonable length
  - Clear input/output
  <!-- Functions should be focused on a single task -->

- Code formatting follows project standards (MUST)
  - Consistent indentation
  - Proper spacing
  - Line length limits respected
  <!-- Code should be consistently formatted according to project standards -->
