# Readability

- **RECOMMENDED:** Ensure complex logic is explained, function purposes are documented, and business rules are clarified for future maintainers.
  - **How to Measure:** Code review and documentation analysis

- **RECOMMENDED:** Ensure all variable and function names reflect their purpose, avoid abbreviations, and follow consistent naming conventions.
  - **How to Measure:** Code review and static analysis tools

- **RECOMMENDED:** Ensure functions follow single responsibility principle, have reasonable length, and have clear input/output.
  - **How to Measure:** Code metrics analysis and review

- **RECOMMENDED:** Ensure consistent indentation, proper spacing, and line length limits are respected according to project standards.
  - **How to Measure:** Run formatter and check for violations
