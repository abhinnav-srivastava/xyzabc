# Code Review Checklist

- Unit tests cover critical business logic (MUST)
  - Core algorithms tested
  - Edge cases covered
  - Test data realistic
  <!-- Essential business logic must have comprehensive unit test coverage -->

- Integration tests verify component interactions (GOOD)
  - API endpoints tested
  - Database interactions verified
  - External service mocks
  <!-- Integration tests ensure components work together correctly -->

- Test coverage meets project requirements (GOOD)
  - Minimum coverage threshold met
  - Critical paths covered
  - Coverage reports reviewed
  <!-- Test coverage should meet or exceed project standards -->
